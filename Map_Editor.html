简介

AlternativaEditor 允许您使用称为道具的特殊 3D 元素为 3D 坦克创建关卡（地图）。道具是完整的 3D 对象或其组成部分，您可以使用它们来创建游戏场景。



道具以库的形式加载到编辑器中。道具可以绕垂直轴旋转。旋转角度可以是用户定义的（任何角度）或 90 度的倍数。

安装

您可以点击此处下载该程序。

该编辑器是作为 Adobe AIR 应用程序制作的。因此，您需要下载并安装 Adobe AIR。

然后，您需要下载并解压 zip 压缩包，并安装名为 AlternativaEditor.air 的文件。安装完成后，您将在桌面上看到一个快捷方式。

编辑器概览

界面

编辑器的界面由几个部分组成：

  • 主菜单；
    用于执行基本操作（导入/导出、编辑、加载道具库、帮助等）
  • 工具栏；
    用于切换主要选项的开启和关闭：
    • 吸附；
    • 物理几何；
    • 网格；
    • 属性窗口；
    • 隐藏道具；
  • 工作区；
  • 道具选择面板：
    • 预览；
    • 道具分为类别和组；
  • 所选道具的属性。



首先，您需要加载道具库。您可以通过转到 Library → Clear and Load 并指定库的根文件夹来完成此操作。



库加载完成后，在右侧的弹出菜单中，您将看到道具的类别和组。



库的大致结构

道具库以文件夹结构呈现，其中包含模型文件 (3DS) 和纹理文件 (JPG) 以及 library.xml 文件。

库中存在以下层次结构：

  • 道具类别（建筑物、围栏、地形等）
    • 道具组 - 复杂元素的组成部分（桥梁、岩石组等）
      • 独立道具

道具概览

几何道具

几何道具描述了关卡的固定几何形状。它们是 3DS 对象，其底座具有矩形横截面。其边长应为网格水平间距（5 米）的倍数。如果道具包含地形坡度，则边界之间的高度差应等于网格垂直间距。每个几何道具都可以包含一个子对象，用于定义与坦克碰撞的几何形状。这种对象应该是一个盒子，其中心位于局部坐标系的原点。盒子的轴应沿其局部坐标系的轴方向。 一个道具可以有多个纹理，您可以在设置菜单中选择。



精灵

为了节省内存，一些道具被表示为精灵——始终面向摄像机的平面对象。例如树木和灌木。

构建地图时，应最后添加精灵。首先，您需要测试主要几何形状，以确保没有问题区域并考虑到所有细微差别。



默认情况下，所有元素都吸附到网格。在精灵和功能元素的情况下，这可能是一个障碍。您可以通过单击工具栏上的磁铁图标 () 将其关闭。精灵可能会在随后的启动中消失。这是由于编辑器中的排序问题。可能会发生它们显示在放置它们的表面下方的情况。

这个问题可以通过以下方式解决：

  1. 在顶部栏中选择；
  2. 选择所有精灵；
  3. 按住 V（垂直移动）的同时拖动它们以找到理想位置。

为了确保这种情况不会发生，您可以为每个高度级别添加几个精灵。然后，您可以按照之前所述调整高度，并简单地复制精灵。正确的保存高度将被保存。执行此操作时，必须关闭吸附功能。



出生点和奖励区域

出生点定义了坦克出现的位置及其朝向。出生点有三种类型：黄色用于死亡混战，蓝色和红色用于指定团队战斗（团队对战、互夺军旗、控制点模式）中两个对立团队坦克出现的位置。出生点必须放置在地面上。



奖励区域指示可能出现奖励的区域，并定义了可能奖励的类型。区域应该是矩形的，其边长是网格水平间距的倍数。掉落物品的类型在属性窗口中定义。奖励区域应位于地面上方，高出 1-2 个间距。此外，建议创建比护甲提升和伤害提升掉落区域多 25% 的速度提升掉落区域。

工作方法

向地图添加道具

  1. 选择您需要添加的道具。您将看到一个绿色半透明光标，显示道具将被放置的位置；
  2. 使用鼠标或键盘上的箭头键将光标移动到所需位置；
  3. 您可以使用 W 和 S 键将道具升高或降低到所需级别；
  4. 可以使用 X 和 Z 键绕其轴旋转道具；
  5. 按空格键放置道具。

添加道具后，光标将变为红色。两个道具不能重叠（占据网格的同一部分）。

您可以选择下一个要添加的元素或编辑当前元素。

编辑地图上的道具

为此，您需要单击该道具。执行此操作后，它将以红色突出显示。现在，使用鼠标和键盘，您可以移动和旋转它，或者通过按 Delete 或 C 将其删除。要复制道具，请在按住 Shift 的同时拖动它。按 K 将道具移动半个段。

此外，您可以为每个道具选择纹理。为此，您需要选择已添加到地图上的道具，打开设置菜单，然后选择纹理。



用于构建关卡地形的瓦片道具通常具有不同的纹理。

如果我们添加了几个相同的道具并将纹理应用于其中一个，您将能够通过按空格键将相同的纹理应用于其他选定的道具。最后选择的纹理将应用于它。

现在再多说一点关于这些纹理的结构。它们都是无缝制作的。因此，它们可以在任何方向上无缝连接。有 1x1 瓦片（5x5 米）以及 2x2（10x10 米）、2x1、3x1、3x2 和 3x3 瓦片。

导航

要移动摄像机，您可以使用鼠标中键。要旋转摄像机，请使用 Alt +中键。使用滚轮进行缩放。当您选择特定道具时，滚轮更敏感，而旋转是在突出显示的对象周围完成的。要返回插入光标而不选择道具，请使用 Ctrl +单击组合。您可以在框架中选择多个对象，确保按住 Ctrl 以免选择光标下的道具并不拖动它。您可以使用 Shift +单击将项目添加到选定项中，或使用 Alt + Shift +单击将其从选定项中删除。

您不能将两个元素放在同一个地方。如果拖动时意外发生这种情况，您将看到一个错误窗口，建议撤消最后一个操作。您还可以通过按 Q 来检查是否有任何元素重叠。冲突的元素将以红色突出显示。您可以使用 CTRL + Z 和 Ctrl + Y 撤消最后一个操作并向前迈出一步。

导出/导入地图文件

您可以随时将地图转换为 XML。您可以在主菜单 File→Export Tanks map 1.x 中或使用 Ctrl + T 执行此操作。导出地图时，您需要指定其名称并添加。xml 格式。

要将地图导入编辑器，请使用 File→Import from tanks 或 Ctrl + T。

在 TanksTestingTool 中测试地图

在测试地图之前，请确保死亡混战模式至少有一个出生点（坦克出现的地方）。

为了测试您的工作结果，您可以在 TanksTestingTool 中打开您的地图。为此，您需要：

  1. 将地图转换为 XML；
  2. 将其命名为 map.xml；
  3. 将您获得的文件移动到带有 TanksTestingTool.swf 文件的库文件夹中；
  4. 启动 TanksTestingTool.swf

如果一切操作正确，地图将显示出来。

细微差别和困难

首先，您需要放置关卡的主要几何形状（地形、建筑物、桥梁等）。之后，您安装精灵（植被）和功能元素（出生点、奖励区域、旗帜等）。

放置出生区域时，请务必确保最大的底盘不会被困在出生点。

在渲染过程中，关卡沿着形成层次结构的道具边界划分为块（关卡 → 大块（建筑物、土地等）→ 每个块中更小的组 → 单个道具）。此时，可能会出现一些困难。

为了避免它们，您需要确保至少有几个“接缝”（道具之间的边界）穿过整个关卡，且不被其他道具中断：



黑线是“接缝”，黄线不是。

地图上的这些接缝越多越好。

如果沿着整个地图穿过有问题建筑物的接缝干扰了其他道具，有两种方法可以解决这个问题：

  1. 用墙壁、块或岩石代替这些建筑物（它们高度有限，在网格限制内，且水平方向很小）；
  2. 尝试沿着这些大型建筑物穿过整个关卡制作接缝。

为了确保层次结构正确，请在坦克测试工具中使用调试模式 (Ctrl + B)（连接成组的道具用绿色标记，有问题的区域用红色标记）。如果某处以红色突出显示，则意味着：

  • 存在之前描述的排序问题；
  • 该位置有坦克或精灵（这就是为什么在构建并测试好主要几何形状后，必须最后添加精灵的原因）。
