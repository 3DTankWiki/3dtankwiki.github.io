Introduction

AlternativaEditor 允许您创建 3D 坦克的等级（地图），使用称为道具的特殊 3D 元素。道具是完整的 3D 对象或其组件部分，您可以用它们来创建游戏场景。



道具在编辑器中以库的形式加载。道具可以绕垂直轴旋转。旋转的角度可以由用户定义（任意角度）或90度的倍数。

安装

您可以下载该程序点击这里.

该编辑器是作为Adobe AIR应用程序制作的。因此，您需要下载并安装Adobe AIR.

然后您需要下载、解压缩一个zip档案，并安装一个名为AlternativaEditor.air的文件。安装后，您将在桌面上看到一个快捷方式。

道具概述

几何道具

几何道具描述了一个等级的固定几何形状。它们是3D对象，底部有一个矩形交叉口部分。它的边长应该是网格水平间隔（5米）的倍数。如果道具包含地形坡度，则边界之间的高度差应该等于网格垂直间隔。每个几何道具可以包含一个子对象，定义与坦克碰撞的几何形状。这种对象应该是一个箱子，箱子的中心位于局部坐标系的原点。箱子的轴应该沿着其局部坐标系的轴方向。一个道具可以有多种纹理供您选择在其上游。设置菜单.



精简

为了节省内存，一些道具被表示为精简的平面物体，这些物体始终被部署到镜头中。例如树木和灌木丛。

在构建地图时，精简应该最后添加。首先，您需要测试主要几何形状以确保其正确性。没有问题区域，并考虑所有细微差别。.



默认情况下，所有元素都会吸附到网格上。这在处理精简和功能性元素时可能会成为一个障碍。您可以通过点击工具栏上的磁铁图标来关闭此功能。）。精简可能在随后的启动中消失。这是由于编辑器中排序的问题。它们可能会显示在所放置表面的下方。

这个问题可以被解决为关注:

  1. 选择在上方的栏中；
  2. 选择所有精简；
  3. 在按住的同时拖动它们V(垂直移动) 找到理想的职位。

为了确保它不会发生，您可以为每个高度等级添加多个精简。然后您可以像之前所述那样调整高度，并简单地复制精简。正确的高度将被保存。在执行此操作时，需要关闭吸附功能。



生成点和奖励动感区域

生成点定义了坦克出现的位置以及它们面朝的方向。有三种类型的生成点：黄色用于死亡混战战斗，蓝色和红色则用于指定两支对立团队的坦克在团队战斗（团队对战、互夺军旗、控制点模式）中出现的位置。生成点必须放置在地面等级上。



Bonus 动感区域表示可能出现奖金的区域，并定义可能奖金的类型。动感区域应为长方形，边长应为网格水平间距的倍数。掉落物品的类型在此定义。属性窗口奖励动感区域应位于地面之上，高出1-2个宇宙。还建议比护甲提升和受损提升区域多创建25%的速度提升掉落动感区域。

工作方法

在地图上添加一个道具

  1. 选择您需要添加的道具。您会看到一个绿色半透明的光标显示道具将被放置的位置；
  2. 使用鼠标或键盘上的箭头键将光标移动到所需的位置;
  3. 您可以使用提升或降低道具到所需的等级W和S钥匙;
  4. 道具可以围绕其轴旋转使用X和Z钥匙;
  5. 按压spacebar放置一个道具。

一旦你添加了一个道具，光标会变成红色。两个道具不能重叠（占据网格的同一部分）。

您可以选择下一个要添加的元素或编辑当前的元素。

在地图上编辑道具

要做到这一点，你需要点击该道具。完成后，它将会变成红色高亮。现在你可以使用鼠标和键盘来移动和旋转它，或者通过按键删除它。Delete或C要复制一个道具，请在按住的同时拖动它Shift. 按压K移动一个道具半个片段。

此外，您可以为每个道具选择一个纹理。要做到这一点，您需要选择一个您已添加到地图中的道具，打开设置菜单，然后选择一个纹理。



用于构建等级地形的瓷砖道具通常具有不同的纹理。

如果我们添加几个相同的道具，并对其中一个应用纹理，您将能够通过按空格键将相同的纹理应用到其他选定的道具上。最后选定的纹理将应用于它。

现在再多说一点关于这些纹理的结构。它们都是无缝制作的。因此，它们可以在任意方向上无缝连接在一起。有1x1瓷砖（5x5米），还有2x2（10x10米），2x1，3x1，3x2和3x3的。

导航

要移动镜头，您可以使用鼠标的中间按钮。要旋转镜头，请使用Alt +中间按钮。使用滚轮进行缩放。当您选择特定道具时，滚轮的灵敏度更高，同时围绕高亮对象进行旋转。要在不选择道具的情况下返回插入光标，请使用Ctrl +单击组合。您可以在一个框架中选择多个对象，确保您按住Ctrl不要选择光标下的道具，也不要拖动它。您可以使用以下方法将项目添加到所选项目中Shift +点击，或者通过使用将其从选定中移除Alt + Shift +点击。

你不能将两个元素放在同一个位置。如果在拖动时意外发生这种情况，你将看到一个错误窗口，建议撤销最后的动作。你还可以通过按下来检查是否有元素重叠。Q冲突的元素将以红色突出显示。您可以撤消最后一个操作并向前一步。CTRL + Z和Ctrl + Y.

导出/导入地图文件

您可以随时将地图转换为 XML。您可以在主菜单的文件→导出坦克地图 1.x 中进行操作，或通过使用Ctrl + T导出地图时，您需要指定其名称并添加.xml格式。

要将地图导入编辑器，请使用文件→从坦克导入或Ctrl + T.

在TanksTestingTool中测试一张地图

在测试你的地图之前，请确保至少有一个重生点（坦克出现的位置）用于死亡混战模式。

为了测试您工作的结果，您可以在TanksTestingTool中打开您的地图。为此您需要：

  1. 将地图转换为XML；
  2. 把它叫做地图.xml;
  3. 将您获得的文件移动到包含TanksTestingTool.swf文件的Library文件夹中；
  4. 启动 TanksTestingTool.swf

如果一切都已正确完成，地图将会显示。

细微之处和困难

首先，您需要放置主要的几何形状（地形、建筑物、路桥大战等）。之后，您安装精简（植被）和功能元素（出生点、奖励动感区域、旗子等）。

放置生成动感区域时，务必确保最大的底盘不会被困在生成点。

在渲染过程中，等级沿着道具的边界被划分为禁止，形成层级（等级 → 大禁止（建筑、土地等）→ 每个禁止中的更小小组 → 个别道具）。此时，可能会出现一些困难。

为了避免它们，你需要确保整个等级中至少有几个“关节”（道具之间的边界），并且这些关节不会被其他道具打断。



黑线是“接头”，黄色线不是。

地图上关节越多越好。

如果贯穿整个地图的沿着问题建筑的关节与其他道具发生冲突，可以通过两种方式解决这个问题：

  1. 用墙代替这些建筑物，禁止，或者石头（高度有限，并且在垂直方向上在网格限制内，水平上相当小）;
  2. 尽量在这些大型建筑沿着整个等级中制造接缝.

为了确保层次结构正确，请使用调试模式（Ctrl + B) 在坦克测试工具中（加入小组的道具用绿色标记，问题地点用红色标记）。如果某些地方被标红，意味着：

  • 之前描述过排序存在问题；
  • 这个地方有一辆坦克或一个精灵（这就是为什么在主几何体已经构建和测试完毕后，必须最后添加精简）。
