WinMTR（网络诊断工具）

Windows：

下载、解压并运行WinMTR.

在“主机”文本箱子中，输入一个服务器的名称，格式为 c[服务器编号] .tankionline.com；例如，第五个服务器的名称将看起来像 c5.tankionline.com。

按下“开始”按钮，然后让程序自行运行至少10分钟，最好是半个小时。在这个期间，最好尝试一下，以确保你能及时发现所有统计问题。

要完成统计数据的收集，请点击“停止”。要将其插入论坛，请点击“复制文本到剪贴板”按钮将结果复制到剪贴板；然后，将数据粘贴到短信中（理想情况下，还可以使用标记 [code=auto:0] .. [/code] 来生成表格查看格式）。

命令提示符

测试延迟:

“延迟” - 在计算机网络中，从客户端到服务器再从服务器返回到客户端传输一个信息包所需的时间。

延迟是服务器连接最重要的特征。它决定了你多快能收到关于其他玩家的当前位置和行动的最新信息，以及你多快能将自己的行动发送到服务器。高且不稳定的延迟常常会导致延迟 - 数据传输的长时间延误。

可以通过使用标准的Windows操作系统来检查延迟。要做到这一点，请运行控制台（命令行）Windows，并输入命令来检查延迟。您可以通过几种不同方式运行命令行。其中一种是点击开始按钮，从菜单中选择“所有程序”，然后查看“附件”。在标准程序列表中将有“命令提示符”。另一种方法是再次点击开始按钮，在打开的窗口中选择运行菜单，在“打开”框中输入‘cmd’并点击确定。最后，最简单的方法是使用键盘组合Win + R。在命令提示符下，输入以下命令。延迟接着是一个用于服务器地址的宇宙。

例如，对第五个服务器的延迟测试将如下所示：延迟 c5.tankionline.com。按下回车键并监控结果。

解释：

当你对服务器进行延迟测试时，结果通常会是这样的；

  • 出站数据包的大小（字节）（默认 - 32 字节）；
  • 响应时间（毫秒）;
  • TTL（生存时间） - 计算机等待服务器响应的时间。

默认情况下，这四个数据包会发送到服务器，然后显示测试结果；

  • 传输的数据包总数；
  • 接收的数据包数量；
  • 丢失的数据包数量；
  • 数据丢失（降解）百分比；
  • 包裹的最高、最低和平均往返时间。

如果您在发送后未收到所有包裹，或者文件交接的最高时间和最低时间之间的差距太大，这可能是网络出现问题的迹象。

有时候，传输数据包（特别是大数据包）的问题是由于自动防病毒软件或防火墙的原因，因此在检查一切之前，最好先在它们的接口上禁用延迟控制。

延迟在0到100之间，文件速度分布在20%（每个其他）的射程内被视为正常。

跟踪路由:有时，有必要了解游戏流量通行证从您到服务器的路线。需要识别该路线上的问题区域，以便在用户/客户端的某些部分发生大规模问题时进行诊断。

您可以使用实用程序 tracert（Windows）或 traceroute（用于 Linux）来完成此操作。程序 tracert，如程序延迟，从命令行界面运行。单击“开始”按钮，选择其中的“所有程序”项（对于早期版本的操作系统为“程序”） - 在标准程序列表中查看“附件”，单击“命令提示符”项。或者单击“开始”按钮，在“打开”窗口中选择“运行”项，输入 cmd 然后按 OK 按钮，或按 Enter。您也可以使用键组合 Win + R。在打开的窗口中，输入 cmdlet tracert c5.tankionline.com（它将追踪到第五个服务器）并按 Enter 键。输入地址。

因此，我们已经建立到终节点的路线。在命令提示符窗口中，在实时跟踪输出中，输入中间节点的名称和IP地址，响应时间将以毫秒显示。

解释：在响应时间最小的路段，传输速度最快。这意味着通道没有过载，并且基本上没有任何显著的干扰。

然而，当响应时间超过标准时，我们会收到一个“请求超时”的结果，这相当于数据包的丢失。

因此，可以确定问题在链条中的位置。如果数据包未到达目的地，这意味着它内部出现了问题。如果链条在中间中断，那么某些中间路由器存在问题。同时，在另一台计算机上，或者在另一条不同的路线上（如果存在），网站可以被访问。如果数据包超过了您的ISP网络，那么问题就出在这里。
